## Format Notes

Markdown is good for clean content structure, version control, and easy website copy editing. It is not ideal when you want precise visual control such as font size, spacing, colors, or page layout.

For font-size editing, better options are:

- HTML: best if this copy will become a web page. You can use classes and CSS for exact font sizes.
- DOCX: best if you want to edit visually in Word or WPS.
- MDX: best if you want Markdown-like writing plus custom React components and typography controls.

Recommended workflow for this portfolio:

- Keep `content-marketing-copy.md` as the source-of-truth copy.
- Use `content-marketing.html` and `styles.css` for final visual font sizes and layout.
